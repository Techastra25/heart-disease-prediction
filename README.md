# Heart Disease Prediction

End-to-end ML classification pipeline on the **Cleveland Heart Disease Dataset**
(UCI ML Repository) — 303 patients, 13 clinical features, binary target
(0 = No Disease, 1 = Disease). Trains and compares Logistic Regression vs
Random Forest, serialises the best model, and deploys real-time predictions
through a Flask REST API.

---

## 🔍 Why This Exists

Real medical data is small and imbalanced — a naive classifier predicting
"No Disease" always still gets decent accuracy, so accuracy alone isn't
enough. This project demonstrates:

- **Stratified split** to preserve class balance across train/test
- **StratifiedKFold(5) CV** — not just a single lucky split result
- **ROC-AUC as the primary metric** — false negatives (missing sick patients)
  are the worst failure mode in clinical screening, not overall accuracy
- **Head-to-head model comparison** before committing to deployment
- **Production Flask API** with `/health`, `/predict`, `/predict/batch`

## 🏗 Architecture
data/heart.csv  (303 rows · 13 clinical features · binary target)
│
▼
┌───────────────────────────────────────────────────────┐
│  src/train.py                                         │
│  1. Load & validate  (null check, type check)         │
│  2. Stratified 80/20 split  →  preserves class ratio  │
│  3. StandardScaler fit on TRAIN ONLY  →  no leakage   │
│  4. Train:                                            │
│       Logistic Regression  │  Random Forest           │
│       max_iter=1000        │  n=100, max_depth=6      │
│  5. Evaluate: accuracy + ROC-AUC + 5-fold Strat-CV   │
│  6. Save best model + scaler  →  models/*.pkl         │
│  7. Generate 4 result charts  →  docs/images/         │
└───────────────────────────────────────────────────────┘
│
▼
┌───────────────────────────────────────────────────────┐
│  src/app.py  (Flask REST API)                         │
│  GET  /health         →  model status + metrics       │
│  POST /predict        →  single patient prediction    │
│  POST /predict/batch  →  N patients, one call         │
└───────────────────────────────────────────────────────┘

## 📊 Actual Results (Reproducible — Not Illustrative)

Generated by `python src/train.py` on `data/heart.csv`. Stored in
`docs/model_results.csv`.
Model                  Accuracy   ROC-AUC   CV Score (5-fold StratifiedKFold)
───────────────────────────────────────────────────────────────────────────
Logistic Regression    95.08%     98.92%    94.39%
Random Forest          85.25%     94.91%    85.80%
→ Best model: Logistic Regression  →  deployed in Flask API

**Classification Report — Logistic Regression (test set · 61 patients):**

precision  recall  f1-score  support

No Disease     1.00    0.89      0.94       28
Disease     0.92    1.00      0.96       33
accuracy                       0.95       61

> **Recall = 1.00 on Disease class = zero false negatives on the test set.**
> In clinical screening, a missed diagnosis is the worst failure mode —
> this is the metric that determines whether a model is deployable.

## 🧹 Preprocessing Decisions (What Interviewers Ask)

| Decision | Why |
|---|---|
| `stratify=y` in split | 303 rows is tiny — one unlucky split skews class ratio |
| Scaler fit on train only | Fitting on full dataset leaks test stats into training, inflating accuracy |
| `StratifiedKFold(5)` for CV | Plain KFold doesn't preserve class ratio across folds |
| Null assertion before training | Corrupt CSV producing clean-looking metrics is a real production failure |

## 🚀 Running It Yourself

```bash
pip install -r requirements.txt

# Train — regenerates models + 4 charts
python src/train.py

# Start API
python src/app.py

# Single prediction
curl -X POST http://localhost:5000/predict \
  -H "Content-Type: application/json" \
  -d '{"age":63,"sex":1,"cp":3,"trestbps":145,"chol":233,
       "fbs":1,"restecg":0,"thalach":150,"exang":0,
       "oldpeak":2.3,"slope":0,"ca":0,"thal":1}'

# Health check
curl http://localhost:5000/health

# Tests
pytest tests/ -v
```
